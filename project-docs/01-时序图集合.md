## 1. 完整端到端时序图

**用途**：设计报告 - 系统架构章节

```mermaid
sequenceDiagram
    autonumber
    participant WebApp
    participant OC as OpenClaw
    participant Qwen as 通义千问 API
    participant WH as Webhook Server
    participant DuoS as Milk DuoS

    Note over WebApp,DuoS: 阶段 1: 情绪提交与分析

    WebApp->>OC: POST /emotion
    Note right of WebApp: {content: "我很开心"}
    activate OC
    
    OC->>Qwen: WebSocket: session.update
    Note right of OC: 配置系统提示词
    activate Qwen
    Qwen-->>OC: 配置确认
    
    OC->>Qwen: WebSocket: conversation.item.create
    Note right of OC: 添加用户情绪文本
    OC->>Qwen: WebSocket: response.create
    Note right of OC: 触发响应
    
    Note over Qwen: AI 情绪分析\n延迟：1-2 秒\n成本：¥0.01/次
    
    Qwen-->>OC: response.text.done
    Note left of Qwen: action_sequence: [1,6,0]\nemotion_detected: "happy"
    deactivate Qwen
    
    OC->>WH: POST /action/milk_duos_001
    Note right of OC: action_sequence: [1,6,0]\ncommand_id: "cmd_abc123"
    activate WH
    WH-->>OC: 200 OK
    Note left of WH: {queued: true}
    deactivate WH
    
    OC-->>WebApp: 200 OK
    Note left of OC: status: "success"\naction_sequence: [1,6,0]\nemotion: "happy"
    deactivate OC
    
    Note over WebApp,DuoS: 阶段 2: 动作推送与执行

    DuoS->>WH: GET /poll/milk_duos_001
    Note right of DuoS: 每 2 秒
    activate WH
    WH-->>DuoS: {commands: []}
    deactivate WH
    
    Note over DuoS: 等待 2 秒
    
    DuoS->>WH: GET /poll/milk_duos_001
    activate WH
    WH-->>DuoS: commands: [{command_id, actions}]
    deactivate WH
    
    Note over DuoS: 执行动作序列\n动作 1: 挥手 (500ms)\n动作 6: 舞蹈 (2000ms)\n动作 0: 站立 (500ms)
    
    DuoS->>WH: POST /ack/cmd_abc123
    Note right of DuoS: {status: "completed"}
    activate WH
    WH-->>DuoS: 200 OK
    deactivate WH
```

## 2. 简化版时序图 (PPT 专用)

**用途**：汇报 PPT - 系统架构页 (非技术评委也能看懂)

```mermaid
sequenceDiagram
    autonumber
    participant User as 用户
    participant WebApp
    participant AI as AI 分析引擎
    participant Robot as 机器人

    Note over User,Robot: 情绪驱动机器人工作流程

    User->>WebApp: 输入情绪文本\n"我今天很开心"
    
    WebApp->>AI: POST /emotion
    
    Note right of AI: AI 实时分析\n识别情绪类型\n生成动作序列\n(1-2 秒)
    
    AI-->>WebApp: 返回动作序列\n[挥手，舞蹈，站立]
    
    WebApp->>Robot: HTTP 推送\n动作指令
    
    Note left of Robot: 轮询获取命令\n每 2 秒检查一次
    
    Robot->>Robot: 执行动作序列\n每个动作间隔 500ms
    
    Robot-->>User: 展示机器人动作
```

    RS->>WS: 关闭连接

    RS-->>OC: 返回分析结果

## 4. Webhook 推送时序图

**用途**：设计报告 - Webhook 模块详细设计章节

```mermaid
sequenceDiagram
    participant OC as OpenClaw
    participant JS as json-webhook-skill
    participant WH as Webhook Server
    participant Q as 命令队列
    participant DuoS as Milk DuoS

    Note over OC,DuoS: 动作推送与队列管理

    OC->>JS: 发送动作序列
    Note right of OC: {action_sequence: [1,6,0]}
    activate JS
    
    JS->>JS: 生成 command_id\n生成 cmd_abc123
    JS->>JS: 添加时间戳
    
    JS->>WH: POST /action/milk_duos_001
    Note right of JS: command_id: "cmd_abc123"\naction_sequence: [1,6,0]\ntimestamp: 1714104000000
    activate WH
    
    WH->>Q: 入队操作
    Note right of WH: threading.Lock 保证线程安全
    activate Q
    Q-->>WH: 入队成功
    deactivate Q
    
    WH-->>JS: 200 OK
    Note left of WH: {queued: true, command_id: "cmd_abc123"}
    deactivate WH
    
    JS-->>OC: 推送成功
    deactivate JS
    
    Note over DuoS,Q: 轮询阶段
    
    DuoS->>WH: GET /poll/milk_duos_001
    WH->>Q: 出队操作 (FIFO)
    Q-->>WH: 获取命令列表
    WH-->>DuoS: {commands: [...]}
    
    Note over DuoS: 执行动作
    
    DuoS->>WH: POST /ack/cmd_abc123
    Note right of DuoS: {status: "completed"}
    WH->>Q: 标记命令完成
    WH-->>DuoS: 200 OK
```

## 5. 轮询机制时序图

**用途**：设计报告 - 硬件轮询模块章节

```mermaid
sequenceDiagram
    participant DuoS as Milk DuoS
    participant WH as Webhook Server
    participant Q as 命令队列

    Note over DuoS,Q: 轮询机制 (Polling Mechanism)

    loop 每 2 秒执行一次
        DuoS->>WH: GET /poll/milk_duos_001
        activate WH
        
        WH->>Q: 查询待执行命令
        Note right of WH: status = "pending"
        activate Q
        Q-->>WH: 返回命令列表
        deactivate Q
        
        alt 有 pending 命令
            WH-->>DuoS: commands: [{command_id, actions}]
            
            Note over DuoS: 执行动作序列
            loop 每个动作
                DuoS->>DuoS: 执行动作\n等待 500ms
            end
            
            DuoS->>WH: POST /ack/{command_id}
            Note right of DuoS: {status: "completed"}
            WH->>Q: 更新状态为 completed
            WH-->>DuoS: 200 OK
        else 无 pending 命令
            WH-->>DuoS: {commands: []}
        end
        
        deactivate WH
        
        DuoS->>DuoS: 等待 2 秒
    end
```

## 6. 错误处理时序图

**用途**：设计报告 - 错误处理章节

```mermaid
sequenceDiagram
    participant WebApp
    participant OC as OpenClaw
    participant Qwen as Qwen API
    participant WH as Webhook Server

    Note over WebApp,WH: 场景 1: AI 分析超时

    WebApp->>OC: POST /emotion
    OC->>Qwen: WebSocket 请求
    activate Qwen
    
    Note over Qwen: 超时 (>5 秒)
    
    Qwen-->>OC: 超时错误
    deactivate Qwen
    OC-->>WebApp: 500 Error
    Note left of OC: error_code: "AI_TIMEOUT"\nmessage: "情绪分析超时"

    Note over WebApp,WH: 场景 2: Webhook 推送失败

    WebApp->>OC: POST /emotion
    OC->>OC: 分析成功
    OC->>WH: POST /action
    activate WH
    
    Note over WH: 服务不可用
    
    WH-->>OC: 503 Service Unavailable
    deactivate WH
    OC->>OC: 重试 1 次
    OC->>WH: POST /action (重试)
    activate WH
    WH-->>OC: 503 Service Unavailable
    deactivate WH
    OC-->>WebApp: 500 Error
    Note left of OC: error_code: "WEBHOOK_FAILED"\nmessage: "推送失败"

    Note over WebApp,WH: 场景 3: ACK 超时

    WebApp->>OC: POST /emotion
    OC->>WH: POST /action
    activate WH
    WH-->>OC: 200 OK
    deactivate WH
    OC-->>WebApp: 200 OK
    
    Note over WH: 等待 ACK (>30 秒)
    
    WH->>WH: 标记命令为 failed\n记录日志
```

## 使用说明

### 渲染 Mermaid 图表

**方式 A：GitHub/GitLab**
- 直接粘贴到 Markdown 文件，自动渲染

**方式 B：VS Code**
- 安装 "Markdown Preview Mermaid Support" 插件
- 预览 Markdown 时自动渲染

**方式 C：在线编辑器**
- 访问 https://mermaid.live
- 粘贴代码，导出 PNG/SVG

**方式 D：PPT 插入**
1. 在 mermaid.live 渲染图表
2. 导出为 SVG 或 PNG
3. 插入 PPT

### 图表配色说明

| 颜色 | 含义 |
|------|------|
| 🔵 蓝色 | 用户/前端层 |
| 🟠 橙色 | AI 分析层 (OpenClaw) |
| 🟣 紫色 | 外部 API (Qwen) |
| 🟢 绿色 | 后端服务 (Webhook) |
| 🔴 红色 | 硬件/执行层 |
