## 情绪强度判断

### 强度修饰词

| 修饰词 | 强度加成 |
|--------|---------|
| 有点、稍微 | +0 (轻度) |
| 很、非常、特别 | +1 (中度) |
| 超级、极其、太...了 | +2 (重度) |

### 示例分析

```
"我有点开心" → happy + 轻度 → [1, 0]
"我很开心" → happy + 中度 → [1, 6, 0]
"我超级开心" → happy + 重度 → [1, 6, 9, 0]
```

## 故事型输入处理

### 提取规则

1. **找情绪词** - 优先提取形容词/副词
2. **找动作词** - 如"挥手"、"跑"等
3. **找上下文** - 场景描述（庆祝、告别等）

### 示例

```
输入："小兔子很开心地跑进森林，看见朋友后挥了挥手"

分析：
- 情绪词："开心" → happy
- 动作词："挥手" → 动作 1
- 场景："看见朋友" → 打招呼场景

输出：[1, 6, 0]
```

## 默认规则

### 无法识别情绪时

```json
{
  "status": "success",
  "action_sequence": [0],
  "action_names": ["站立"],
  "emotion_detected": "unknown",
  "rationale": "未检测到明确情绪，返回待机动作"
}
```

### 矛盾情绪时

取**第一个**或**最强**情绪：

```
"我又开心又难过" → 取第一个 → happy → [1, 6, 0]
```

## 动作序列生成算法

```
1. 识别情绪类型 (emotion)
2. 判断情绪强度 (intensity: 0/1/2)
3. 查表获取基础序列 (base_sequence)
4. 根据强度调整序列长度
5. 确保以 [0] 结尾 (回归待机)
6. 输出最终序列
```

### 伪代码

```python
def generate_sequence(emotion, intensity):
    base = EMOTION_MAP.get(emotion, [0])
    
    if intensity == 0:
        return base[:2]  # 轻度：取前 2 个
    elif intensity == 1:
        return base      # 中度：完整序列

        return base + [9, 0]  # 重度：加前进动作
```
